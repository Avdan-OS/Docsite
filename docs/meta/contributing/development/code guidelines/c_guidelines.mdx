## Names

#### Use pascal case for structs and snake case for variables and functions.

:::tip

When defining a const variable always write it in UPPER CASE

:::

**Do ‚úÖ**:

```c
struct Example;
size_t example_t;
void example_func();
int example_var;
const EXAMPLE_VAR;
```

**Don't ‚ùå**:

```c
struct example;
size_t exampleT; // size_t exampleSize;
void ExampleFunc(); // void exampleFUNC();
int exampleVar;
const example_var;
```

:::caution

When declaring a costant use always CONST and not PREPROCESSOR DIRECTIVES

:::

#### Use full words, except when the variable name is too long.

**Do ‚úÖ**:

```c
int my_var;
int length;
char width;
```

**Don't ‚ùå**:

```c
int my_variable;
int len;
char width;
```

#### Use descriptive names in functions.

**Do ‚úÖ**:

```c
char convert_to_ascii(int);
```

**Don't ‚ùå**:

```c
char to_ascii(int);
```

#### Leave useless variable names out of function declarations.

**Do ‚úÖ**:

```c
char my_character(char);
```

**Don't ‚ùå**:

```c
char my_character(char character);
```

## Headers

#### Use ```#pragma once``` for header files. Why? Because ```#pragma once``` is less prone to making mistakes and it is less code to type.

**Do ‚úÖ**:

```c
#pragma once
```

**Don't ‚ùå**:

```c
#ifndef FILE_H
#define FILE_H

// code

#endif // FILE_H
```

#### When including headers files, include first the standard libraries and then local headers

**Do ‚úÖ**:

```c
#pragma once

#include <stdio.h>
#include <stdlib.h>

#include "file1.h"
#include "file2.h"

// code
```

**Don't ‚ùå**:

```c

#pragma once

#include "file1.h"
#include <stdio.h>
#include "file2.h"

#include <stdlib.h>

// code
```

## Structs

#### Always use ```typedef struct``` for structs.

**Do ‚úÖ**:

```c
typedef struct {
    int a;
    int b;
} MyStruct;
```

**Don't ‚ùå**:

```c
struct MyStruct {
    int a;
    int b;
};
```

## Functions

#### When writing a function, or a for cycle, don't write curly braces at the beginning of a new line.

**Do ‚úÖ**:

```c

int my_function(int a, int b) {
    // code
}

for (int i = 0; i <= 10; i++) {
    printf("%d\n", i);
}
```

**Don't ‚ùå**:

```c
int my_function(int a, int b)
{
    // code
}

for (int i = 0; i <= 10; i++)
{
    printf("%d\n", i);
}
```

## Statements

#### Use conditional operators instead of if else statements.

**Do ‚úÖ**:

```c
my_variable % 2 == 0 ? printf("even") : printf("odd");
```

**Don't ‚ùå**:

```c
if (my_variable % 2 == 0) {
    printf("even");
} else {
    printf("odd");
}
```

#### When the statement is too long, go to a new line.

:::tip

Same thing as above, but for a simple line of code.

:::

**Do ‚úÖ**:

```c
my_variable  % 2 == 0 && my_variable2 % 4 == 0 ? printf("foo") : \
my_variable3 % 6 == 0 && my_variable4 % 8 == 0 ? printf("fee") : printf("fuu");
```

**Don't ‚ùå**:

```c
my_variable  % 2 == 0 && my_variable2 % 4 == 0 ? printf("foo") : my_variable3 % 6 == 0 && my_variable4 % 8 == 0 ? printf("fee") : printf("fuu");
```

## Types

#### Delete ```int``` when using ```unsigned``` modifier, and don't use ```signed``` modifier.

**Do ‚úÖ**:

```c
unsigned my_variable;
int my_variable2;
```

**Don't ‚ùå**:

```c
unsigned int a;
signed b;
signed int c;
```

## Comments

#### Always remember to add comments to your code to make it easier to read.

**Do ‚úÖ**:

```c
// This function does this thing
unsigned my_func(unsigned a, unsigned b) {
    // code
}
```

**Don't ‚ùå**:

```c
unsigned my_func(unsigned a, unsigned b) {
    // code
}
```

#### Be coerent when you write comments

**Do ‚úÖ**:

```c
// This function does this thing
unsigned my_func(unsigned a, unsigned b) {
    // code
}

// This function does this thing 2
unsigned my_func2(unsigned a2, unsigned b2) {
    // code
}
```

**Don't ‚ùå**:

```c
//this function does this thing.
unsigned my_func(unsigned a, unsigned b) {
    // code
}

//  this function does this thing 2
unsigned my_func2(unsigned a2, unsigned b2) {
    // code
}
```

#### When writing big comments use ```/* */``` instead of ```//```

**Do ‚úÖ**:

```c
/*
 * This
 * is
 * a
 * very
 * big
 * comment
 */
```

**Don't ‚ùå**:

```c
// This
// is
// a
// very
// big
// comment
```
