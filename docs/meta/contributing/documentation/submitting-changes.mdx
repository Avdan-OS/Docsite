> So, you you want to write documentation? hmmm...

## 1. Create a fork
To get started, fork [our docs repo](https://github.com/Avdan-OS/docsite) on GitHub.

## 2. Clone your fork
Clone your fork to your local machine.

```bash title="Cloning your fork"
git clone https://github.com/<your-username>/docsite.git
```

## 3. Make some changes
Write some documentation to your fork.

:::tip
Make sure to keep your fork up to date with the latest changes by periodically syncing it with ours.
:::
:::tip
If you use VS Code, you can use our snippets to help you along the way.
:::

## 4. Commit and push your changes
Commit and push your changes to your fork.

## 5. Submit a pull request
Submit a pull request to Avdan-OS/docsite repo.
We'll get round to reviewing your changes and if they fit our guidelines we'll merge them into the main repo.

## 6. See your changes on the site
If all goes well, your docs will be live on this very site. How cool is that?
